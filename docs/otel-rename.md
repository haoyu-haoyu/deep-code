Rationale for hard cutover

- DeepCode is self-use only — no external Grafana/Datadog dashboards depend
on the old names.
- DeepSeek-TUI (Hmbown/DeepSeek-TUI) has no OTel infrastructure and provides
no dual-emit precedent.
- OpenAI Codex CLI uses its own naming (codex.*) from scratch.
- Dual-emit doubles metric volume and adds code without benefit when no
consumer reads the old names.

Perf-gate transition

The CI perf gate (.github/workflows/ci.yml) compares head.json vs base.json
by `metrics[].label`. The first PR that does the rename has head with
deepcode_* labels and base with unprefixed labels → no overlap → no
comparisons. The gate may
silently pass with zero comparisons.

The P1.10 rename PR uses one of:

- Option a (chosen): add a temporary mapping layer in
`packages/deep-code/scripts/perf-compare.mjs` that compares old base labels to
new head labels by first applying namespace renames (`claude_code.` → `deepcode.`,
`com.anthropic.claude_code.` → `ai.deepcode.`), then applying the
perf-baseline label rule (`<label>` → `deepcode_<label>` for unprefixed
baseline labels such as `jsonl_parse_1k_msgs_ms`). Remove the mapping layer
in a follow-up PR one merge later.
- Option b: skip the perf gate for the rename PR with an explicit comment in
the PR body, restored in the next PR.
