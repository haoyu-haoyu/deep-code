OpenTelemetry rename plan

Status: Decided
Decision date: 2026-05-10

Summary

Hard cutover from claude_code.* and com.anthropic.claude_code.* to
deepcode.* and ai.deepcode.*. No dual-emit window.

| Old | New |
|---|---|

| com.anthropic.claude_code.* | ai.deepcode.* |

Rationale for hard cutover

- DeepCode is self-use only — no external Grafana/Datadog dashboards depend
on the old names.
- DeepSeek-TUI (Hmbown/DeepSeek-TUI) has no OTel infrastructure and provides
no dual-emit precedent.
- OpenAI Codex CLI uses its own naming (codex.*) from scratch.
- Dual-emit doubles metric volume and adds code without benefit when no
consumer reads the old names.

perf-baseline JSON keys

| Old | New |
|---|---|
| jsonl_parse_1k_msgs_ms | deepcode_jsonl_parse_1k_msgs_ms |
| (all other perf-baseline keys) | (deepcode_ prefix added) |

Perf-gate transition

The CI perf gate (.github/workflows/ci.yml) compares head.json vs base.json
keys. The first PR that does the rename has head with deepcode_* keys and
base with unprefixed keys. No overlap means no comparisons. The gate may
silently pass with zero comparisons.

The P1.10 rename PR uses one of:

- Option a (chosen): add a temporary mapping layer in
scripts/perf-compare.mjs that translates base keys via the
claude_code to deepcode rule when comparing, removed in a follow-up PR
one merge later.
- Option b: skip the perf gate for the rename PR with an explicit comment in
the PR body, restored in the next PR.

Tests required before P1.10

- Counter creation: STATE.sessionCounter is created with name
deepcode.session.count, not claude_code.session.count.
- No string occurrence of claude_code. (with trailing dot) in
packages/deep-code/src/ after rename.
- No string occurrence of com.anthropic.claude_code. in
packages/deep-code/src/ after rename.
- perf-baseline JSON output keys all start with deepcode_.

Phase 1 unblock

This decision unblocks P1.10 (strip GrowthBook/Statsig + OTel rename).
